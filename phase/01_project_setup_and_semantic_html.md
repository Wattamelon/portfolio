# Phase 01. 프로젝트 골격과 시맨틱 HTML

## 이 단계의 목표
이 단계에서는 포트폴리오 사이트의 뼈대를 먼저 만듭니다.
화면이 예쁘게 보이는 것보다, 전체 구조가 명확하게 나오는 것이 우선입니다.

## 미션에서 충족할 요구
- `index.html` 중심의 기본 프로젝트 구조 만들기
- `header`, `nav`, `main`, `section`, `footer` 같은 시맨틱 태그 사용
- `Hero`, `About`, `Skills`, `Projects`, `Contact`, `Footer` 섹션 구성
- `label`, `input`, `button`, `a`, `img` 같은 기본 요소를 올바르게 배치
- `alt`와 폼 연결 등 기본 접근성 요소 반영

## Codex에게 요청할 작업
1. 프로젝트 폴더 구조를 정리한다.
2. `index.html`에 사이트 전체 섹션을 먼저 만든다.
3. 각 섹션의 제목과 기본 콘텐츠를 채운다.
4. 네비게이션 링크가 각 섹션의 `id`와 연결되도록 한다.
5. 프로필 이미지, 프로젝트 카드, 문의 폼의 자리만 우선 배치한다.
6. JavaScript가 나중에 붙을 수 있도록 필요한 `id`, `class`, `data-*` 속성을 미리 준비한다.

## 왜 이 단계가 필요한가
HTML 구조가 먼저 잡혀야 CSS와 JavaScript가 어디에 붙을지 정해집니다.
처음부터 스타일이나 기능부터 만들면 나중에 구조를 바꾸느라 작업이 꼬이기 쉽습니다.

## 이 단계가 끝나면 나오는 결과
- 브라우저에서 섹션들이 순서대로 보이는 페이지
- 메뉴 클릭 시 각 섹션으로 이동 가능한 기본 구조
- 이후 CSS와 인터랙션을 붙이기 쉬운 상태

## 완료 기준
- 페이지에 최소한 아래 섹션이 모두 존재한다.
  - Header
  - Hero
  - About
  - Skills
  - Projects
  - Contact
  - Footer
- 섹션마다 구분 가능한 `id`가 있다.
- 폼 요소와 이미지에 기본 접근성 속성이 들어 있다.

## Codex 출력 기대값
- 구조가 깔끔한 `index.html`
- 이후 단계에서 수정하기 쉬운 시맨틱 마크업
