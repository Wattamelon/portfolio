# Phase 02. 기본 스타일과 반응형 레이아웃

## 이 단계의 목표
이 단계에서는 HTML 뼈대에 스타일을 입혀서, 데스크톱과 모바일 모두에서 보기 좋게 만듭니다.

## 미션에서 충족할 요구
- 외부 라이브러리 없이 순수 CSS로 스타일 구성
- `Flexbox`와 `Grid`로 레이아웃 배치
- CSS 변수로 색상, 간격, 폰트 통일
- 반응형 구성 구현
- 모바일, 태블릿, 데스크톱에서 레이아웃이 자연스럽게 바뀌도록 처리

## Codex에게 요청할 작업
1. `css/style.css` 또는 기존 스타일 파일을 만든다.
2. 전체 색상, 간격, 폰트, 그림자, 라운딩을 CSS 변수로 정의한다.
3. `body`, `header`, `section`, `button`, `card`의 기본 스타일을 정리한다.
4. `Hero`, `About`, `Skills`, `Projects`, `Contact` 영역에 맞는 레이아웃을 만든다.
5. `@media`를 사용해 모바일에서는 세로 흐름, 데스크톱에서는 넓은 흐름으로 바꾼다.
6. 카드와 버튼에 기본적인 hover, transition, box-shadow 효과를 넣는다.

## 왜 이 단계가 필요한가
미션의 핵심은 "반응형 포트폴리오"입니다.
기능을 먼저 붙여도, 레이아웃이 무너지면 실제 제출 결과가 좋지 않습니다.

## 이 단계가 끝나면 나오는 결과
- 기본적으로 보기 좋은 포트폴리오 화면
- 화면 크기에 따라 자연스럽게 바뀌는 레이아웃
- 다음 단계에서 인터랙션을 얹기 쉬운 시각적 기반

## 완료 기준
- 모바일에서 가로 스크롤이 거의 없고 읽기 편하다.
- 데스크톱에서 섹션들이 넓게 정돈되어 보인다.
- 카드와 버튼의 기본 시각 효과가 적용되어 있다.
- CSS 변수와 공통 스타일이 잘 정리되어 있다.

## Codex 출력 기대값
- 반응형 레이아웃이 적용된 CSS
- 반복 스타일을 줄인 유지보수 가능한 스타일 구조
