# Phase 03. 내비게이션과 UI 인터랙션

## 이 단계의 목표
이 단계에서는 사용자가 페이지를 실제로 조작할 수 있게 만듭니다.
메뉴 열기/닫기, 부드러운 이동, 스크롤 버튼, 테마 전환 같은 기본 인터랙션을 구현합니다.

## 미션에서 충족할 요구
- 햄버거 메뉴 토글
- 부드러운 스크롤 이동
- 스크롤 탑 버튼
- 다크 모드 또는 테마 전환
- `addEventListener`, `classList.toggle`, `preventDefault()` 사용

## Codex에게 요청할 작업
1. `js/main.js` 또는 동등한 스크립트 파일을 연결한다.
2. 모바일에서 햄버거 버튼을 누르면 네비게이션이 열리고 닫히게 만든다.
3. 메뉴 링크를 클릭하면 해당 섹션으로 부드럽게 이동하게 처리한다.
4. 페이지 아래로 내려가면 스크롤 탑 버튼이 보이게 만들고, 누르면 맨 위로 이동하게 한다.
5. 다크 모드 토글 버튼이 있다면 `localStorage`에 상태를 저장하고 새로고침 후에도 유지되게 한다.
6. 스크롤 위치에 따라 헤더 스타일이 바뀌는 연출이 필요하면 추가한다.

## 왜 이 단계가 필요한가
이 미션은 "정적인 페이지"가 아니라 "반응형 포트폴리오 웹사이트"입니다.
즉, 사용자가 직접 움직일 수 있어야 하고, 그 동작이 DOM과 연결되어야 합니다.

## 이 단계가 끝나면 나오는 결과
- 모바일에서 메뉴를 열고 닫을 수 있는 페이지
- 메뉴 클릭 시 자연스럽게 이동하는 페이지
- 스크롤 탑 버튼으로 편하게 맨 위로 돌아갈 수 있는 페이지
- 테마 전환이 있으면 사용자가 모드를 바꿀 수 있는 페이지

## 완료 기준
- 햄버거 메뉴가 정상 동작한다.
- 메뉴 클릭 시 기본 이동이 방해되지 않고 부드럽게 스크롤된다.
- 스크롤 탑 버튼의 노출/비노출이 자연스럽다.
- 필요한 상태가 DOM 클래스 변화로 잘 표현된다.

## Codex 출력 기대값
- 이벤트 기반 인터랙션이 들어간 JavaScript
- 사용자 행동에 즉시 반응하는 UI
