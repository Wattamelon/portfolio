# Phase 04. GitHub API 연동과 프로젝트 카드 렌더링

## 이 단계의 목표
이 단계에서는 GitHub API에서 프로젝트 데이터를 가져와서 Projects 섹션에 자동으로 보여줍니다.

## 미션에서 충족할 요구
- `fetch`와 `async/await` 사용
- GitHub API 호출
- 응답 데이터를 카드 형태로 렌더링
- 로딩, 성공, 실패, 빈 데이터 상태 처리
- `map`, `filter`, `forEach` 같은 배열 메서드 활용

## Codex에게 요청할 작업
1. GitHub 사용자 이름 또는 저장소 설정값을 분리한다.
2. `fetch`로 저장소 목록을 가져오는 함수를 만든다.
3. 받아온 데이터를 Projects 카드로 변환한다.
4. 별 개수, 설명, 링크, 언어 등 표시할 항목을 정리한다.
5. 요청 중에는 로딩 상태를 보여주고, 실패 시 에러 메시지를 보여준다.
6. 데이터가 비어 있으면 빈 상태 UI를 보여준다.
7. 필요하면 특정 조건으로 프로젝트만 골라서 보여준다.

## 왜 이 단계가 필요한가
미션 요구사항 중 핵심 중 하나가 외부 API 연동입니다.
이 과정을 통해 정적인 자기소개 페이지가 아니라, 실제 데이터가 살아 있는 포트폴리오가 됩니다.

## 이 단계가 끝나면 나오는 결과
- GitHub 저장소 목록이 자동으로 표시되는 Projects 섹션
- API 상태에 맞는 피드백 UI
- 실제 데이터 기반의 동적 포트폴리오

## 완료 기준
- API 호출이 성공하면 카드 목록이 보여진다.
- 호출 실패 시 사용자에게 에러 메시지가 보인다.
- 데이터가 없을 때도 화면이 깨지지 않는다.
- 렌더링이 한 번에 끝나지 않더라도 로딩 상태가 안내된다.

## Codex 출력 기대값
- GitHub API 연동 코드
- 상태별 UI를 포함한 프로젝트 카드 렌더링 로직
