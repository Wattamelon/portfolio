# Phase 05. Contact 폼 유효성 검사와 UX

## 이 단계의 목표
이 단계에서는 Contact 폼이 단순한 입력칸이 아니라, 실제로 사용할 수 있는 폼이 되도록 만듭니다.

## 미션에서 충족할 요구
- Contact 섹션에 이름, 이메일, 메시지 폼 존재
- 필수값 검증
- 이메일 형식 검증
- 에러 메시지 표시
- 제출 시 기본 동작 제어
- `event.preventDefault()` 활용

## Codex에게 요청할 작업
1. 폼 필드별로 검증 규칙을 정한다.
2. 빈 값, 이메일 형식 오류, 메시지 길이 부족 같은 케이스를 검사한다.
3. 제출 버튼을 눌렀을 때 유효하지 않으면 에러 메시지를 띄운다.
4. 유효하면 성공 메시지를 보여주거나 다음 동작으로 넘긴다.
5. 입력 중에도 필요하면 실시간 피드백이 가능하도록 한다.
6. 화면을 해치지 않게 에러 UI를 정리한다.

## 왜 이 단계가 필요한가
폼은 사용자가 직접 정보를 넣는 구간이라서 실수 가능성이 큽니다.
검증이 없으면 불친절하고, 배포 후에도 신뢰도가 떨어집니다.

## 이 단계가 끝나면 나오는 결과
- 잘못된 입력을 막는 Contact 폼
- 사용자가 무엇을 고쳐야 하는지 바로 알 수 있는 화면
- 제출 흐름이 안정적인 문의 영역

## 완료 기준
- 빈 입력으로 제출하면 막힌다.
- 이메일 형식이 틀리면 안내가 나온다.
- 에러 메시지가 입력 필드 근처에 명확하게 표시된다.
- 정상 입력 시 성공 동작이 보인다.

## Codex 출력 기대값
- 검증 로직이 포함된 Contact 폼 처리 코드
- 사용성까지 고려된 에러/성공 메시지 UI
