# Phase 06. 접근성, 최종 다듬기, 배포

## 이 단계의 목표
이 단계에서는 전체 완성도를 높이고, 실제 제출 가능한 상태로 마무리합니다.

## 미션에서 충족할 요구
- 접근성 점검
- 키보드 조작 가능 여부 확인
- `alt`, `label`, 버튼 역할 등의 기본 접근성 보강
- README에 프로젝트 설명, 사용 기술, 배포 URL, 스크린샷 정리
- GitHub Pages 배포

## Codex에게 요청할 작업
1. 탭 이동만으로도 사이트를 사용할 수 있는지 확인한다.
2. 이미지, 폼, 버튼, 링크의 접근성을 점검한다.
3. 색 대비와 폰트 가독성을 마지막으로 확인한다.
4. 필요한 경우 반복되는 스타일과 JS를 정리한다.
5. README에 프로젝트 설명, 실행 방법, 배포 URL, 캡처 이미지를 정리한다.
6. GitHub Pages 배포용 경로와 파일 참조를 점검한다.

## 왜 이 단계가 필요한가
기능이 돌아가도, 접근성이 약하거나 배포 후 경로가 깨지면 미션 결과가 좋지 않습니다.
마지막 단계는 "실제로 제출 가능한 완성본"으로 만드는 작업입니다.

## 이 단계가 끝나면 나오는 결과
- 실제 제출 가능한 최종 포트폴리오 사이트
- 배포 링크가 동작하는 상태
- README까지 정리된 완성형 프로젝트

## 완료 기준
- 키보드만으로 주요 기능을 사용할 수 있다.
- 기본 접근성 속성이 빠지지 않았다.
- 배포된 URL에서 화면이 정상 동작한다.
- README가 프로젝트를 이해하기 충분하다.

## Codex 출력 기대값
- 최종 정리된 코드
- 배포와 제출을 위한 마무리 문서
